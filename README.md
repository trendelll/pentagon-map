# Pentagon Fragility Map: Frontend Documentation

This repository contains the Stage 3 Visualization Dashboard for the Pentagon Fragility project. It is built using Next.js 15, Tailwind CSS, and TypeScript.

## Architecture: The Headless Design
To ensure high performance and ease of collaboration, the frontend uses a decoupled architecture. It does not query the PostgreSQL database directly. Instead, it consumes the Stage 3 JSON export generated by the K-Medoids pipeline.

* Data Bridge: data/cluster_output.json
* Benefits: This allows the UI to render instantly on any machine (Windows or Mac) without requiring a local database setup or the 1.8GB raw data archive.

## Directory Map

```text
src/
├── app/                  # Next.js App Router (File-based routing)
│   ├── category/[id]     # Deep-dive pages for Aviation, Maritime, etc.
│   ├── contractor/[id]   # Market footprint for companies (e.g., Lockheed)
│   └── page.tsx          # Main Global Overview (The "Map")
├── components/           # Reusable UI elements
│   ├── FragilityBadge.tsx    # Maps K-Medoid labels to UI status
│   └── PercentageBarChart.tsx # Visualizes market share dominance
└── lib/
    └── data.ts           # TypeScript interfaces for cluster data

    
    
    
    Key Components and Data Flow
1. The Data Interface (lib/data.ts)

Specific TypeScript interfaces match the output of the PAM (Partitioning Around Medoids) script. This ensures that if the clustering logic changes, such as modifying the k-value, the UI remains stable.
2. Fragility Mapping (FragilityBadge.tsx)

This component translates the K-Medoid cluster archetypes into visual risk levels:

    Extreme Concentration / Monopoly Risk: Assigned high-alert visual styling.

    Moderate Concentration / Mixed: Assigned moderate-alert visual styling.

3. Market Share Visualization (PercentageBarChart.tsx)

This component renders the top_contractors array for each category. It specifically highlights the Parent UEI Rollup logic, showing how a single prime contractor dominates various Product and Service Codes (PSCs).
How to Run

Since the frontend is configured to read the cluster_output.json file, the dashboard can be started with the following steps:

    Install dependencies:
    npm install

    Start the development server:
    npm run dev

    View the app:
    Open http://localhost:3000 in your browser.
